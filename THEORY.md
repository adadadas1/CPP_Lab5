## Классы и структуры в C++

**Класс (class)** — тип данных, объединяющий данные (поля) и поведение (методы). По умолчанию члены класса `private`.
**Структура (struct)** — то же самое, но члены по умолчанию `public`. В C++ разница только в доступе по умолчанию.

```cpp
class Ball {
public:
    void setVelocity(const Velocity& velocity);  // публичный интерфейс
    Velocity getVelocity() const;
private:
    Velocity m_velocity;  // инкапсулированное состояние
    Point    m_center;
    double   m_radius = 1.0;
};
```

### Инкапсуляция

Приватные поля скрыты от внешнего кода. Доступ только через публичные методы. Это позволяет изменить внутреннюю реализацию, не ломая код, который использует класс.

## Геттеры и сеттеры

Типичный паттерн для контролируемого доступа к полям:

```cpp
void Ball::setRadius(double radius) {
    // можно добавить валидацию, не меняя интерфейс
    m_radius = radius;
}

double Ball::getRadius() const {
    return m_radius;
}
```

`const` после параметров метода означает, что метод не модифицирует объект (можно вызывать на `const Ball&`).

## Перегрузка оператора ввода

Чтобы читать объект из потока через `>>`:

```cpp
std::istream& operator>>(std::istream& stream, Point& point) {
    stream >> point.x >> point.y;
    return stream;
}
```

После этого работает:
```cpp
Point p;
std::cin >> p;         // читает два числа
stream >> p1 >> p2;   // цепочка чтения
```

В проекте это используется в конструкторе `World` для чтения шаров из файла.

## Принцип единственной ответственности (SRP)

Каждый класс отвечает за одну вещь:

| Класс | Ответственность |
|-------|-----------------|
| `Ball` | данные одного шара (позиция, скорость, радиус, цвет) |
| `World` | список шаров, загрузка модели, оркестрация |
| `Physics` | физические вычисления (перемещение, коллизии) |
| `Painter` | рисование примитивов на экране |
| `Color` | представление цвета (RGB) |
| `Point` | 2D-координата |
| `Velocity` | вектор скорости |

## Делегирование отрисовки через Painter

```cpp
void Ball::draw(Painter& painter) const {
    painter.draw(m_center, m_radius, m_color);
}
```

Ball не знает, как рисовать на экране — он только знает *что* нарисовать (круг с данными параметрами). Конкретный механизм рисования — ответственность Painter. Это пример **инверсии зависимостей** (DIP).

## Дискретная симуляция физики

Физика симулируется дискретными "тиками". На каждом тике шары смещаются на `velocity * timePerTick` и проверяются коллизии:

```cpp
void World::update(double time) {
    // Время разбивается на тики длиной timePerTick = 0.001 с
    // Остаток накапливается в restTime
    time += restTime;
    const auto ticks = static_cast<size_t>(std::floor(time / timePerTick));
    restTime = time - double(ticks) * timePerTick;
    physics.update(balls, ticks);
}
```

`restTime` позволяет не терять дробную часть времени между вызовами `update`.

## Флаг isCollidable

Шары с `isCollidable == false` проходят сквозь другие объекты. Это позволяет создавать "призрачные" шары для визуальных эффектов, не нарушая физику основных.

```cpp
class Ball {
public:
    bool isCollidable() const;
    void setIsCollidable(bool value);
private:
    bool m_isCollidable = true;
};
```

## Краткая сводка для защиты

### Ключевые определения

- **Класс** — пользовательский тип данных: поля (данные) + методы (поведение). Члены `private` по умолчанию.
- **Инкапсуляция** — скрытие внутренней реализации за публичным интерфейсом.
- **Геттер/сеттер** — методы для чтения/записи приватного поля с возможностью добавить логику.
- **`const` метод** — не изменяет объект (`this` — константный указатель). Можно вызывать на `const` объекте.
- **SRP (Single Responsibility Principle)** — каждый класс отвечает за одну вещь.
- **Перегрузка `operator>>`** — позволяет читать объект из потока через `stream >> obj`.
- **Делегирование** — класс передаёт часть работы другому объекту (Ball делегирует рисование Painter).
- **Дискретная симуляция** — физика считается не непрерывно, а шагами фиксированной длины (тиками).

### Типичные вопросы на защите

**Q: В чём разница `class` и `struct` в C++?**
A: Только в доступе по умолчанию: `class` — `private`, `struct` — `public`. Всё остальное одинаково.

**Q: Зачем использовать геттеры/сеттеры вместо публичных полей?**
A: Позволяют добавить валидацию, логирование, вычисляемые значения без изменения интерфейса. Поле можно сделать `readonly` (только геттер).

**Q: Почему `getRadius()`, `getMass()` — `const` методы?**
A: Они не изменяют состояние шара. `const` позволяет вызывать их на `const Ball&`, что требуется в `World::show`.

**Q: Как реализован `getMass()`?**
A: `M_PI * r * r * r * 4.0 / 3.0` — формула объёма шара (масса пропорциональна объёму при единичной плотности).

**Q: Почему Ball не рисует себя сам (без Painter)?**
A: Разделение ответственностей (SRP). Ball не должен знать про API рисования — это позволяет поменять рендерер без изменения Ball.

**Q: Что такое `restTime` в World::update?**
A: Накопленный остаток времени, который не был обработан в предыдущем вызове update. Без него симуляция теряла бы дробные части тиков.

**Q: Зачем `isCollidable`?**
A: Шары с false проходят сквозь других — полезно для временных визуальных эффектов (частицы после столкновения).
